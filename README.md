## ğŸ— Data Modeling

The project follows a **Star Schema** design for clear, performant analytics:

- **Fact Tables:**
  - `fact_atliqo_metrics`
  - `fact_market_share`
  - `fact_plan_revenue`

- **Dimension Tables:**
  - `dim_date`
  - `dim_cities`
  - `dim_plan`

**Key Relationships:**

- `dim_date[date]` â†” `fact_atliqo_metrics[date]`  
- `dim_date[date]` â†” `fact_plan_revenue[date]`  
- `dim_date[date]` â†” `fact_market_share[date]`  
- `dim_cities[city_code]` â†” all fact tables `[city_code]`  
- `dim_plan[plan]` â†” `fact_plan_revenue[plans]`

The **`before_after_5g`** column in `dim_date` is used extensively to create pre vs post comparisons and time-matching logic.

> This model enables flexible slicing by **city**, **plan**, **time period**, and **company** while keeping the model simple and efficient.
